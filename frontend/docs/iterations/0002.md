## What I added

A **Dark / Light toggle** in the top bar (moon/sun icon, sitting next to the Accessibility and Monday/Weekday controls), so the owner can switch to the dark "Command Center" look from the sample HTML he liked.

- **`src/index.css`** — added a `.dark` block that overrides the neutral theme tokens (background, card, muted, border, foreground) with a deep teal-slate canvas mirroring the client's dark mockup, plus brightened brand (`primary`/`brand`), semantic (`success`/`warning`/`destructive`), and chart hues so everything stays legible on dark surfaces. Because every component reads these CSS variables, the whole app — all six tabs, panels, status pills, bars, gradient banner — re-themes instantly with no per-component changes.
- **`src/components/AppShell.tsx`** — a small `DarkToggle` that flips the `.dark` class on `<html>`.

It composes cleanly with the existing **Accessibility** color-vision selector: the color-blind palettes apply their semantic/chart overrides as inline styles on top of either light or dark, so both controls work together. Default remains light (so the tuned clinical brand is the first impression), with dark one click away.

`pnpm build` (tsc + vite) passes clean. Kept to the two files the change required.
