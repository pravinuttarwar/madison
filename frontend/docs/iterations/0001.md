## What I added

There was no color-vision control before — the prototype was built color-blind-*safe* (status shown by icon + shape + label, not color alone) but didn't expose a way to pick a palette. I added an **Accessibility** selector in the top bar (eye icon, next to the Monday/Weekday toggle).

Opening it shows a labeled menu with three options, each with a live swatch preview and a one-line note:
- **Default palette** — standard clinical colors
- **Red–green friendly** — deuteranopia/protanopia; "good" reads blue instead of green
- **Blue–yellow friendly** — tritanopia; avoids amber/blue collisions

Selecting one retunes the semantic (`--color-success/warning/destructive`) and chart (`--color-chart-1..5`) theme tokens on `:root`, so every status pill, trend badge, and bar across all six tabs updates instantly. The brand identity (primary teal / blue) stays fixed. The menu copy reinforces the accessibility design and the "we'd validate the exact palette with you" discovery framing.

Implementation notes:
- Self-contained `ColorVision` component in `AppShell.tsx`; closes on outside click; keyboard/ARIA roles (`menuitemradio`, `aria-checked`).
- No persistence layer touched (no raw storage), so it follows CLAUDE.md conventions and stays focused.
- `pnpm build` (tsc + vite) passes clean.
